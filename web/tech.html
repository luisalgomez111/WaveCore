Documentación Técnica

Bienvenido a la guía de ingeniería de WaveCore. Este documento detalla la infraestructura, decisiones de diseño y el stack tecnológico que sustenta nuestra plataforma de gestión de audio de alto rendimiento.

¿Qué es WaveCore?

WaveCore es una aplicación de escritorio diseñada para profesionales del sonido (editores de video, diseñadores de FX, músicos) que necesitan gestionar librerías masivas con velocidad quirúrgica. A diferencia de un explorador de archivos común, WaveCore indexa metadatos técnicos y genera visualizaciones de onda al instante.

Stack Tecnológico

Hemos seleccionado tecnologías que equilibran la rapidez de desarrollo con el rendimiento en tiempo real:

Lógica: Python 3.12

Utilizado para la gestión de la "Bóveda" (Vault), la manipulación de la base de datos local y la comunicación con APIs externas (Freesound).

Interfaz: PyQt6

Qt6 permite una renderización fluida del visor de ondas y una gestión de hilos (Threads) eficiente para que la interfaz nunca se bloquee durante la indexación.

Audio: FFmpeg & SoundFile

FFmpeg se encarga de la decodificación de formatos complejos (WAV, MP3, OGG, FLAC) y la extracción de datos de amplitud para el visor profesional.

Arquitectura Modular

WaveCore sigue un patrón de diseño desacoplado para asegurar la estabilidad:

UI LAYER (PyQt6)

⇅ (Signals & Slots)

CORE LOGIC (Python Threads)

⇅ (Direct I/O)

HARDWARE (FFmpeg Engine / SSD)

Este sistema permite que mientras el motor de audio procesa un archivo, el usuario pueda seguir buscando otros sonidos sin latencia.

Guía de Compilación

Para construir WaveCore desde el código fuente, utiliza nuestro script de build optimizado:

# Instalar dependencias pip install -r requirements.txt # Ejecutar script de empaquetado python build_exe.py

Este proceso genera un ejecutable `.exe` portátil que incluye todas las dependencias y el motor FFmpeg necesario para su funcionamiento.

CI/CD y Despliegue

WaveCore utiliza una infraestructura moderna de integración continua. Cada "Push" al repositorio de GitHub activa una acción de automatización:

  • Web: Se despliega automáticamente en Vercel para mantener la documentación siempre actualizada.
  • App: Se preparan los logs de cambios y se validan las rutas de los instaladores generados con Inno Setup.
